Key Design Decisions

A

Monorepo Structure

backend/ · frontend/ · firmware/ · configs/ · tests/ — single repo, clear boundaries

B

YAML Configuration

All settings in configs/*.yaml — zero hardcoded values, hot-reloadable

C

WebSocket Communication

Laptop ↔ ESP32 via JSON over WebSocket — low latency, bidirectional

D

Phased Development

7 phases, each building on the last — scaffold → capture → AI → action → servo → dashboard → optimize
